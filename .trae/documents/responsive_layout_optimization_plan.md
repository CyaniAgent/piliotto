# PiliOtto - 响应式布局优化计划

## [x] 任务 1: 使用Flutter的内置特性

* **优先级**: P0

* **依赖关系**: 无

* **描述**:
  * 利用AspectRatio、FittedBox等组件处理比例和缩放
  * 使用LayoutBuilder实时获取父容器尺寸
  * 充分利用Flexible和Expanded创建自适应布局

* **成功标准**:
  * 代码更加简洁，利用框架特性而非手动计算
  * 布局更加稳定，适应各种屏幕尺寸

* **测试要求**:
  * `programmatic` TR-1.1: 布局在各种屏幕尺寸下正确显示
  * `human-judgement` TR-1.2: 代码简洁，易于理解

* **备注**: 逐步替换手动计算的布局逻辑，使用Flutter内置组件

## [x] 任务 2: 提取适配工具类

* **优先级**: P0

* **依赖关系**: 任务 1

* **描述**:
  * 创建ResponsiveUtil工具类，封装常用的响应式计算逻辑
  * 提供统一的平台检查方法
  * 封装常用的布局计算方法

* **成功标准**:
  * 工具类能够被各个页面使用
  * 减少代码重复，提高代码可维护性

* **测试要求**:
  * `programmatic` TR-2.1: 工具类方法正确计算布局参数
  * `human-judgement` TR-2.2: 工具类使用方便，接口清晰

* **备注**: 工具类应该包含屏幕尺寸判断、间距计算、列数计算等常用功能

## [x] 任务 3: 统一响应式布局策略

* **优先级**: P0

* **依赖关系**: 任务 2

* **描述**:
  * 定义统一的响应式断点常量
  * 使用MediaQuery和LayoutBuilder获取屏幕尺寸
  * 统一处理不同屏幕尺寸的布局逻辑

* **成功标准**:
  * 所有页面能够根据屏幕尺寸自动调整布局
  * 断点定义清晰，便于维护

* **测试要求**:
  * `programmatic` TR-3.1: 不同屏幕尺寸下布局正确显示
  * `human-judgement` TR-3.2: 布局在各种屏幕尺寸下看起来美观

* **备注**: 重点关注首页、推荐页、热门页等主要页面的响应式布局

## [x] 任务 4: 组件化改造

* **优先级**: P1

* **依赖关系**: 任务 3

* **描述**:
  * 将常用的布局模式封装为可复用组件
  * 为不同平台提供统一的组件接口
  * 组件内部处理平台差异

* **成功标准**:
  * 代码复用率提高
  * 组件接口清晰，易于使用

* **测试要求**:
  * `programmatic` TR-4.1: 组件在不同平台上正确显示
  * `human-judgement` TR-4.2: 组件使用方式简洁明了

* **备注**: 重点关注搜索栏、视频卡片、列表项等常用组件

## [/] 任务 5: 状态管理优化

* **优先级**: P1

* **依赖关系**: 任务 4

* **描述**:

  * 使用GetX的响应式特性，减少手动状态管理

  * 集中管理与屏幕尺寸相关的状态

  * 优化状态更新逻辑，避免不必要的重建

* **成功标准**:

  * 状态管理清晰，易于维护

  * 状态更新及时，UI响应迅速

* **测试要求**:

  * `programmatic` TR-5.1: 状态更新正确反映到UI

  * `human-judgement` TR-5.2: 状态管理逻辑清晰易懂

* **备注**: 重点关注响应式布局相关的状态管理
